#### 프롤로그
```
* 평소 전자정부프레임워크(스프링 부트 - 일부 차이점이 있지만, 해당 내용은 다른 파일에서 작성하도록 하겠습니다.) 를 사용하고 있어,
  컴파일, 빌드 등 기본적인 과정에 대한 생각을 하지 않고 작업했다. 
  예전부터 "개발이 제일 중요하지 그런게 뭐가 중요해?" 이렇게 생각하는 부족한 개발자 마인드 였기에,
  생각 자체를 바꾸고자 한다. (멍청한 자식)
```

#### 컴파일(Compile)
```
"컴파일러는 특정 프로그래밍 언어로 쓰여 있는 문서를 다른 프로그래밍 언어로 옮기는 언어 번역 프로그램을 말한다"  
```
> 즉 사람이 작성한 언어를 기계가 읽을 수 있도록 번역하는 것이 컴파일입니다.  
> 개발자가 작성한 소스코드를 바이너리 코드로 변환하는 과정이다.  
> 자바에서는 컴파일을 할 경우, JVM에서 실행 가능한 바이트코드 형태의 클래스 파일이 생성된다.  
> (.java라는 자바 클래스 파일을 바탕으로 .class라는 클래스 파일이 생성)

#### 빌드(Build)
```
"빌드는 소스 파일을 컴퓨터나 휴대폰에서 실행할 수 있는 독립 소프트웨어 가공물로 변환하는 과정을 말하거나, 그에 대한 결과물을 일컫는다"
```
> 즉 문서나 리소스등을 하나로 합쳐 앱으로 실행되게 하는 과정이나 앱이다.  
> 빌드란 소스코드 파일들을 컴퓨터에서 실행할 수 있는 소프트웨어로 변환하는 일련의 과정이다  
> 컴파일, 테스팅, 배포 등 소스코드 파일을 실행 가능한 소프트웨어 산출물로 만드는 일련의 과정을 말한다.

#### 빌드 도구(Build Tool)
> 빌드 도구는 빌드 과정을 자동으로 수행해주는 도구를 의미한다.  
> 일반적으로 빌드 툴이 제공해주는 기능  
> → 전처리(preprocessing), 컴파일(Compile), 패키징(packaging), 테스팅(testing), 배포(distribution)  
> 빌드 도구 예 : Ant, Maven, Gradle 
```
* Ant는 빌드 도구로만 이용 되었는데, 빌드와 자동 라이브러리 관리 기능이 추가된 Maven이 등장하였다.
  현재 재직중인 회사에서도, 메이븐을 적극 활용하고 있다.
  Maven 이후에 Gradle이 나왔는데, 요즘은 Gradle로 넘어가는 추세라 한다. (왜 우리회사는...)
  Gradle은 안드로이드 앱의 공식 빌드 시스템이다.
  (Gradle 공식 문서에 따르면 Gradle이 Maven보다 최대 100배 빠르다고 한다.)
```
