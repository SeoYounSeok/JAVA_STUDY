### 추상클래스와 템플릿 메서드

```
프레임 워크에서 많이 사용되는 설게 패턴
로직 흐름을 정의하는 역할
예)  public final void run() {
        startCar();
        drive();
        stop();
        stopCar();
    }

추상 클래스로 선언된 상위 클래스에 템플릿 메서드를 활용하여 전체적인 흐름을 정의하고 하위 클래스에 맞게 구현
```

#### 추상클래스의 역할

```
1. 추상 클래스는 하위 클래스 에서도 사용할 수 있는 코드를 구현
2. 시나리오를 정의할 수 있는 메서드를 구현해 놓으면 시나리오 대로 진행
```

#### 템플릿 메서드

```
* 추상메서드나 구현된 메서드를 활용하여 전체 기능의 흐름(시나리오)를 정의하는 메소드
  예) 예제 함수의 run();

1. 전체 기능의 흐름(시나리오)을 정의하는 메서드
2. final 선언하면 하위 클래스에서 재정의할 수 없음.
```

#### final

```
final 변수는 값이 변경될 수 없는 상수이다.
예) public static final double PI = 3.14;

final 변수는 오직 한 번만 값을 할당할 수 있다.
final 메서드는 하위 클래스에서 재정의 할 수 없다.
final 클래스는 더 이상 상속되지 않는다.(JAVA의 String 클래스)
```
