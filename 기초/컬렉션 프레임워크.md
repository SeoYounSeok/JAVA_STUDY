#### 제너릭 프레임워크

```
- 변수의 선언이나 매서드의 매개변수를 하나의 참조 자료형이 아닌 여러 자료형을 변환할 수 있도록 프로그래밍하는 방식
- 실제 사용되는 참조 자료형의 변환을 컴파일러가 검증하는 안정적인 프로그래밍
예) 리스트의 타입을 넣지 않고, add 를 하면, object 로 인식하기 때문에 제너릭이 필요하다.
- 컬렉션 프레임워크에서 많이 사용되고 있다.

- 여러 참조 자료형으로 대체될 수 있는 부분을 하나의 문자로 표현
- 이 문자를 자료형 매게변수 라고 한다.

type 의 의미에서 T 를 많이 사용한다.
static 키워드는 T 에 사용할 수 없다.
```

#### 컬렉션 프레임워크

```
- 프로그램 구현에 필요한 자료구조를 구현해 놓은 라이브러리
- java.util 패키지에 구현되어 있다.
- 개발에 소요되는 시간을 절약하면서 사용방법을 이해해야 한다.
- 여러 인터페이스와 구현 클래스 사용 방법을 이해해야 한다.

* 컬렉션 인터페이스
- 하나의 객체를 관리하기 위한 메서드가 정의된 인터페이스
- 하위에 List 와 Set 인터페이스가 있다.
- 여러 클래스들이 Collection 인터페이스를 구현한다.

1. List 인터페이스 : 순서가 있는 자료 관리, 중복 허용
2. Set 인터페이스 : 순서가 정해져 있지 않다. 중복을 허용하지 않는다.
3. Map 인터페이스
- 쌍으로 이루어진 객체를 관리하는데 사용한다.
- Key-value 의 쌍으로 이루어져 있다.
- Key 는 중복될 수 없다.
- 여러 클래스들이 Map 인터페이스를 구현한다.
```
